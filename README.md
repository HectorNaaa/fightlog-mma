# FightLog MMA

This is a [Next.js](https://nextjs.org) project bootstrapped with [`create-next-app`](https://nextjs.org/docs/app/api-reference/cli/create-next-app).

## Getting Started

First, run the development server:

```bash
npm run dev
# or
yarn dev
# or
pnpm dev
# or
bun dev
```

Open [http://localhost:3000](http://localhost:3000) with your browser to see the result.

You can start editing the page by modifying `app/page.tsx`. The page auto-updates as you edit the file.

This project uses [`next/font`](https://nextjs.org/docs/app/building-your-application/optimizing/fonts) to automatically optimize and load [Geist](https://vercel.com/font), a new font family for Vercel.

## Environment Setup

Copy `.env.example` to `.env.local` and set:

- `DATABASE_URL`
- `DIRECT_URL` (if your provider requires a direct connection string)
- `JWT_SECRET` (must be at least 32 chars)
- `NEXT_PUBLIC_APP_URL`

If `JWT_SECRET` is missing or too short, login/signup will fail by design with a clear configuration error.

## Database

Run:

```bash
npm run db:push
npm run build
```

## Auth Reliability Checklist

- Ensure `JWT_SECRET` is defined in local and Vercel environments.
- Ensure `DATABASE_URL` points to a reachable Postgres database.
- Verify auth routes:
  - `POST /api/auth/signup`
  - `POST /api/auth/login`
  - `POST /api/auth/logout`
  - `GET /api/auth/me`
- Dashboard routes are protected via middleware.
- Auth pages redirect authenticated users to `/dashboard`.
